## 🎬 촬영 순서 (8장 세트)

스토리텔링 순서로 구성했습니다. 사용자가 스와이프하며 "이 앱이 내 문제를 해결해주겠다"고 확신하도록.

### #1 홈 대시보드 — "한눈에 보는 축산 사업장"
**촬영 화면**: `HomeScreen` (홈 탭)
**포함 요소**:
- 상단에 사업장명·오늘 날짜
- 오늘의 매출 / 마진 / 재고 현황 카드
- 위생점검 미완료 알림
- 바로가기 버튼들
**상단 헤드라인 오버레이**: `한눈에 보는 축산 사업장`

---

### #2 위생일지 — "HACCP 점검, 탭 한 번으로 끝"
**촬영 화면**: `HygieneScreen`
**포함 요소**:
- 위생관리 체크리스트 일부 O로 체크된 상태
- 하단 "전체선택 (O)" 버튼 노출
- 오늘 날짜 상단 표기
**상단 헤드라인 오버레이**: `HACCP 점검, 탭 한 번으로 끝`

---

### #3 숙성관리 — "드라이에이징, 실패 제로"
**촬영 화면**: `AgingScreen`
**포함 요소**:
- 숙성 중인 고기 3~4건 목록
- 시작일, 경과일, 중량 변화, 수율
- "D-5", "D-14", "D-21" 숙성 단계 표시
**상단 헤드라인 오버레이**: `드라이에이징, 실패 제로`

---

### #4 마감 정산 — "오늘 마진, 3초 만에 확인"
**촬영 화면**: `ClosingScreen` 또는 `MarginScreen`
**포함 요소**:
- 오늘 매출 / 원가 / 마진 큰 숫자로 표시
- 부위별 판매 그래프
- "PDF 출력" 버튼 노출
**상단 헤드라인 오버레이**: `오늘 마진, 3초 만에 확인`

---

### #5 이력번호 조회 — "바코드 한 번이면 끝"
**촬영 화면**: `ScanScreen` 결과 화면
**포함 요소**:
- 실제 이력 조회 결과(도축일/등급/출생지)
- 축종·부위·등급 명시
- "저장" 버튼 노출
**상단 헤드라인 오버레이**: `바코드 한 번이면 끝`

---

### #6 AI 서류 스캔 — "촬영만 하면 자동 입력"
**촬영 화면**: `UploadScreen` OCR 결과
**포함 요소**:
- 좌측: 서류 이미지 썸네일
- 우측: 자동 추출된 텍스트 필드들 (사업자번호, 상호 등)
- "AI 자동 인식 완료" 배지
**상단 헤드라인 오버레이**: `촬영만 하면 자동 입력`

---

### #7 구독 플랜 — "출시 기념 180일 무료"
**촬영 화면**: `PaywallScreen`
**포함 요소**:
- 🎉 "출시 기념 · 전 플랜 무료" 빨간 배너 크게
- "180일 남음" 카운트다운
- 베이직/프로 플랜 비교 카드
**상단 헤드라인 오버레이**: `180일 전 플랜 무료 이벤트`

---

### #8 구인구직 — "검증된 축산 인력 매칭"
**촬영 화면**: `JobOwnerBrowseScreen`
**포함 요소**:
- 구직자 프로필 카드 3~4개
- 지역·경력·희망급여 필터 드롭다운 노출
- A/B 등급 배지
**상단 헤드라인 오버레이**: `검증된 축산 인력 매칭`

## 🛠 촬영 방법

### 방법 1: 실기기 촬영 (권장)

**iPhone**:
1. 미트빅 앱 설치 후 프로덕션 빌드 실행
2. **볼륨↑ + 전원** 동시 짧게 누르기
3. `사진` 앱에서 편집 → 크기 확인 (1290×2796 or 1242×2688)
4. AirDrop으로 맥 or 메일로 PC 전송

**Android**:
1. APK 설치 후 실행
2. **볼륨↓ + 전원** 동시 누르기 (또는 손날로 쓸어내리기)
3. `파일 관리 > Pictures/Screenshots` 에서 확인
4. USB 또는 Google Drive로 PC 전송

### 방법 2: Xcode Simulator (맥 필요)
- iPhone 16 Pro Max 시뮬레이터 실행
- `Cmd + S` 로 스크린샷 자동 저장
- 실기기 없이도 정확한 해상도 획득 가능

### 방법 3: Android Emulator
- Android Studio > AVD Manager > Pixel 7 Pro
- 에뮬레이터 툴바의 카메라 버튼 클릭

## 🎯 체크리스트 (최종 제출 전)

### 각 스크린샷마다
- [ ] 실제 기능이 작동하는 화면이 맞는가 (더미·플레이스홀더 NO)
- [ ] 개인정보(실제 사업자번호, 본명, 전화번호) 노출 없음
- [ ] 시간 표시는 모두 같은 시각(예: 오전 9:41)으로 통일 — Apple 관행
- [ ] 배터리 풀, Wi-Fi 연결 상태로 통일
- [ ] 푸시 알림·팝업 없는 깨끗한 상태
- [ ] 헤드라인 문구가 폰트 깨짐 없이 표시
- [ ] 글씨 크기가 썸네일에서도 읽히는가

### 8장 전체
- [ ] 순서가 스토리텔링 논리대로 배치 (#1 → #8)
- [ ] 색상·폰트·구성이 일관됨
- [ ] Android와 iOS 스크린샷이 동일 내용·순서

## 🚀 업로드 순서

### Play Store
1. `Play Console > 스토어 등록정보 > 그래픽`
2. "전화기 스크린샷" 섹션에 8장 드래그&드롭

### App Store
1. `App Store Connect > 앱 > iOS 버전 > iPhone 6.9" 디스플레이`
2. "+" 버튼으로 8장 업로드
3. 드래그로 순서 조정
4. 다른 크기(6.5", 5.5")는 같은 이미지 재사용 가능

## 💡 프로 팁

### 1. Frame Up 도구로 디바이스 프레임 입히기
- 단순 스크린샷보다 **실제 폰에 들어있는 것처럼** 보이게
- 도구: https://deviceframes.com (무료) / https://previewed.app (무료)
- 특히 App Store는 프레임 없는 순수 스크린샷만 허용하니 **Play Store용**으로만 활용

### 2. A/B 테스트 준비
- 헤드라인 2가지 버전 만들어두고, 출시 후 1주일 단위로 교체 실험
- Play Console의 "Store Listing Experiments" 기능으로 데이터 기반 최적화

### 3. 현지화 고려 (영문 버전은 나중에)
- 초기에는 한국어만 등록
- 해외 확장 시 영문 스크린샷 세트 별도 제작
- Pretendard는 영문도 예쁘게 나옴
